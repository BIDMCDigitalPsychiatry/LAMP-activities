# Water Sort — Data Dictionary

## Cognitive Background

The Water Sort puzzle is a planning and problem-solving task in the same cognitive family as the Tower of London (Shallice, 1982) and Tower of Hanoi. Participants sort colored blocks across tubes so that each tube contains only one color. The task engages **executive function** — specifically planning, working memory, and inhibitory control — by requiring multi-step lookahead under move constraints.

### Task Design

- **Tubes** with capacity 4 blocks each; 2 extra empty tubes for workspace
- **Colors**: 3 (easy), 4 (medium), or 5 (hard) distinct colors
- Only the **top block** on a tube can be moved
- A block can only be placed on a **matching color** or into an **empty tube**
- When pouring, all consecutive same-color blocks on top transfer together (as many as fit)
- Puzzles are generated by reverse-shuffling from the solved state, guaranteeing solvability
- Minimum pours are computed via BFS at generation time
- A **per-puzzle time limit** prevents indefinite stalling

### Key Metrics

- **Planning time** (first-pour latency): time from puzzle display to first pour — reflects pre-planning
- **Execution time**: time from first pour to solution — reflects plan execution
- **Excess pours**: actual pours minus minimum pours — reflects plan quality
- **Puzzles solved in minimum**: the primary accuracy measure

## Settings

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `difficulty` | string | "medium" | Difficulty level: "easy" (3 colors), "medium" (4 colors), "hard" (5 colors) |
| `max_puzzles` | number | 8 | Number of puzzles to present |
| `time_limit_per_puzzle_s` | number | 90 | Time limit per puzzle in seconds |

## `static_data` Fields

| Field | Type | Description |
|-------|------|-------------|
| `score` | number | % puzzles solved in minimum pours (legacy compat, 0–100) |
| `correct_answers` | number | Puzzles solved in minimum pours (legacy compat) |
| `total_questions` | number | Total puzzles attempted (legacy compat) |
| `max_puzzles` | number | Max puzzles setting |
| `time_limit_per_puzzle_s` | number | Time limit setting |
| `difficulty` | string | Difficulty setting |
| `puzzles_attempted` | number | Total puzzles shown |
| `puzzles_solved` | number | Puzzles solved (any pour count) |
| `puzzles_solved_in_minimum` | number | Puzzles solved in exactly the minimum pours |
| `total_excess_pours` | number | Sum of (actual − minimum) across solved puzzles |
| `mean_excess_pours` | number | Average excess pours per solved puzzle |
| `mean_planning_time_ms` | number | Mean first-pour latency across all puzzles |
| `mean_execution_time_ms` | number | Mean execution time across solved puzzles |
| `puzzles_timed_out` | number | Puzzles where time ran out |
| `questionnaire` | {clarity, happiness} | Post-game ratings (1–5 each) |

## `temporal_slices` Fields

Each entry represents one puzzle (not individual pours).

| Field | Type | Description |
|-------|------|-------------|
| `item` | string | Puzzle identifier (e.g., "puzzle_3") |
| `type` | string | "solved", "timeout", or "exit" |
| `puzzle_number` | number | Puzzle number (1-indexed) |
| `num_colors` | number | Number of colors in this puzzle |
| `minimum_pours` | number | BFS-computed optimal solution length |
| `actual_pours` | number | Pours the participant made |
| `excess_pours` | number | actual_pours − minimum_pours |
| `solved` | boolean | Whether all tubes were sorted |
| `planning_time_ms` | number | Time from display to first pour (ms) |
| `execution_time_ms` | number | Time from first pour to solution (ms) |
| `duration` | number | Total time for this puzzle (ms) |
| `pour_sequence` | string[] | Sequence of pours (e.g., ["0→2", "3→1"]) |
| `level` | number | Number of colors (difficulty proxy) |
